# 학생 작품모음 패들렛 푸시 알림 설치방법

## 파일 구성

- `Code.gs`: Google Apps Script 서버 코드
- `index.html`: GitHub Pages에 올릴 앱 화면
- `sw.js`: Web Push 서비스워커
- `manifest.json`: PWA 설치 정보
- `icon-192.png`, `icon-512.png`: 앱 아이콘

## 1. Apps Script 배포

1. 기존 학생 작품모음 Apps Script의 `Code.gs` 전체를 이 폴더의 `Code.gs` 내용으로 교체합니다.
2. Apps Script에서 배포를 새로 만듭니다.
   - 유형: 웹 앱
   - 실행 사용자: 나
   - 액세스 권한: 모든 사용자
3. 배포 후 `/exec`로 끝나는 웹 앱 URL을 복사합니다.

## 2. GitHub Pages 파일 설정

1. `index.html`을 열어 아래 줄을 찾습니다.

```js
const SCRIPT_URL = 'PASTE_APPS_SCRIPT_EXEC_URL_HERE';
```

2. 따옴표 안을 Apps Script 웹 앱 `/exec` URL로 바꿉니다.
3. GitHub 저장소 루트에 아래 파일을 올립니다.
   - `index.html`
   - `sw.js`
   - `manifest.json`
   - `icon-192.png`
   - `icon-512.png`
4. GitHub Pages를 켭니다.
5. GitHub Pages 주소가 `https://rlasksk030.github.io/student-padlet-alarm/`가 아니라면 `Code.gs`의 `PWA_APP_URL`도 실제 주소로 바꿉니다.

## 3. Apps Script에서 한 번 실행

처음에는 Apps Script 편집기에서 아래 함수를 순서대로 직접 실행합니다.

1. `testPadletFetchFirstStudent()`
2. `setupPadletPostAlertTrigger()`

`testPadletFetchFirstStudent()`가 성공해야 자동 감지가 가능합니다. 만약 “Padlet 보안 확인 화면” 오류가 나오면 Padlet이 서버 자동 확인을 막는 상태라서, 별도 우회 방식이 필요합니다.

## 4. 부모님 사용 방법

1. GitHub Pages 앱 주소를 엽니다.
2. 학생 이름과 핀번호로 로그인합니다.
3. iPhone은 공유 버튼에서 “홈 화면에 추가” 후 홈 화면 앱으로 다시 엽니다.
4. 상단의 `알림` 버튼을 누르고 권한을 허용합니다.
5. `테스트` 버튼으로 잠금화면 알림이 오는지 확인합니다.

## 작동 방식

`setupPadletPostAlertTrigger()`를 실행하면 현재 패들렛 상태를 먼저 저장합니다. 그래서 기존 글에는 알림을 보내지 않고, 이후 새 글 또는 내용 변화가 감지될 때만 해당 학생으로 등록된 기기에 푸시 알림을 보냅니다.
